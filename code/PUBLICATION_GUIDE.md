# Publication Guide

This repository is a research prototype based on Move-r. The current branch
contains the original Move-r baselines plus an Vr-RLZSA variant
used for the paper experiments.

## What To Commit

Recommended public files:

- Core algorithm changes:
  - `include/move_r/move_r.hpp`
  - `include/move_r/queries.cpp`
  - `cli/move-r/move-r-build.cpp`
  - `cli/move-r/move-r-locate.cpp`
  - `cli/move-r/move-r-partial-codec.cpp`
  - `cli/move-r/move-r-dump-fields.cpp`
  - `CMakeLists.txt`
- Experiment scripts:
  - `measurements/run_selected_m4_128.py`
  - `measurements/run_innovation2_varint_einstein_100mb.py`
  - `measurements/run_innovation2_varint_v2_cross_dataset.py`
  - `measurements/run_vrrlzsa_ablation.py`
  - `measurements/dump_vrrlzsa_field_frequency.py`
- Documentation:
  - `CURRENT_ALGORITHM_DESCRIPTION.md`
  - `INNOVATION2_EXPERIMENTS.md`
  - `PUBLICATION_GUIDE.md`
  - `paper_results/innovation2/`

Do not commit local datasets, generated indexes, raw pattern files, build
directories, or full experiment logs. They are ignored by `.gitignore`.

## Public Result Bundle

The compact result bundle for paper writing is under:

```text
paper_results/innovation2/
```

It contains:

- `varint_v2_cross_dataset/`: cross-dataset comparison for Move-r,
  Move-r-RLZ, Move-r-RLZEnd, r-RLZSA@25%, and r-RLZSA@12.5%.
- `vrrlzsa_ablation/`: raw partial RLZSA vs Vr-RLZSA-v1 vs Vr-RLZSA-v2.
- `field_frequency_top100/`: compact top-100 rank-frequency data for SR/CPL
  fields, plus per-dataset field summaries.

The original full outputs remain in `measurements/results/` locally, but that
directory is intentionally ignored because it contains large logs and generated
intermediate data.

## Build

Typical build on Ubuntu 24.04:

```bash
mkdir -p build
cd build
cmake ..
make -j
```

The new tools generated by the current algorithm are:

- `build/cli/move-r/move-r-build`
- `build/cli/move-r/move-r-locate`
- `build/cli/move-r/move-r-partial-codec`
- `build/cli/move-r/move-r-dump-fields`

## Reproduce Vr-RLZSA

Place datasets under `measurements/texts/` using the names expected by the
scripts, then run:

```bash
python3 measurements/run_selected_m4_128.py
python3 measurements/run_innovation2_varint_v2_cross_dataset.py
python3 measurements/run_vrrlzsa_ablation.py
python3 measurements/dump_vrrlzsa_field_frequency.py
```

For a smaller sanity check:

```bash
python3 measurements/run_innovation2_varint_einstein_100mb.py
```

The scripts write full generated outputs to `measurements/results/`, which is
ignored. Copy only compact summaries into `paper_results/` when preparing a
paper artifact or release.

## Notes For Reviewers

`Move-r-RLZ` is the rlz-only baseline in the experiments. It still stores the
Move-r structures required for backward search, but the locate phase uses the
full RLZSA representation of the differential suffix array.

`r-RLZSA@25%` and `r-RLZSA@12.5%` keep RLZSA data only for selected SA blocks.
The selected blocks are chosen from training queries by coverage score, and
non-selected blocks fall back to Phi-based Move-r locate.
